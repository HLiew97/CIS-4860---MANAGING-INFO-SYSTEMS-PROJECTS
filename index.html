Emotion Recognition AI

Build an AI model for recognizing facial expressions and determining emotional responses. This model will take webcam data and, in real time, analyze the emotion of a person captured by the camera. I chose this AI-focused project because I haven't built an AI model before, and I believe it's a crucial skill for the future.

📏 High-Level Scope

▼

✅ In Scope for Semester Project

  • Determine which AI base model is most appropriate for this project
  • Secure training data to teach the model
  • Train the model off the training data and have it develop pattern recognition
  • Test the model with data not within the training dataset
  • Develop a presentation for end of class reporting to the professor

❌ Out of Scope

  • Detecting subtle emotions such as "guilt" is more complex and most likely not accomplish-able in the duration of the semester.
  • Alternate Culture emotion detection can be difficult to achieve, as different groups of people can react differently when expressing emotion.
  • Long-term data collection is ethically questionable for the purposes of this educational project.
  • Integrating this model into wearable devices like smart glasses is a task too outside the requirements of just creating the AI.

🗺️ Roadmap Items (≤6 Months)

Upload the model to allow usage beyond my personal server
Enhance the model with real data to refine its algorithms
Leverage the project for usage in my resume and job applications
Consider further applications and features that can be expanded on to create a more complex AI

🧩 Project Methodology Explanation

▼

Based on my analyses, I believe that a Hybrid Iterative approach would be the best methodology for my project. This is because the development of the AI requires an iterative refinement as I add more training data and run the model against testing data. In addition, the more definite pathway of a hybrid approach is better for the project so that I can align mid project tasks to the end goal. Having a traditional approach is somewhat too rigid for me because I am also learning as I complete the project, so I cannot foresee everything that the project will encounter. Finally, the product should be mostly feature complete immediately and then refined over iterations, so an incremental approach doesn't really work here.
