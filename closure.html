üß≠ Executive Summary

‚ñº

Overall, the project was successful. The technical process of obtaining the data and model ran into some issues with running it locally. Using an alternative in Google Colab to perform the project on allowed me to bypass the issue. While the robustness of the final version of the model is not where I would like it to be, the core functionality of the model works. Model currently only really works with pictures that are 48x48 pixels and only in grayscale. Accuracy is decent, with a roughly 80% chance of correctly determining a pictures‚Äô emotion. If allowed more time to iterate the model, it may be able to achieve a higher accuracy rate and allow for adding robustness to the model.

üîé Retrospective

‚ñº

What Went Well

  • Feedback from stakeholders helped shape the project towards a successful version.
  • Project planning allowed the focus of the project to stay on how to improve the model, rather than ambiguously deciding next steps.

What Didn't Go Well

  • Trying to find meaningful progress each week was difficult when parts of the project were focused on research and learning.
  • Scope was slightly too large for the timeline of the semester, with additional improvements failing to be met.
  • Technical aspect of the project was somewhat underestimated, with model installation proving more difficult than expected.

What Would You Do Differently

  • I would create a better timeline with buffers for unexpected issues.
  • Having alternative plans made before executing would be helpful as a fallback in the case of technical troubles.

üìå Key Takeaways

‚ñº
  • Allow buffer time for technical considerations when designing the timeline of the project.
  • Schedule workflow ahead of time to better facilitate project execution.
  • Allow flexibility in scope of project but determine what absolutely needs to be completed to consider the project complete.
